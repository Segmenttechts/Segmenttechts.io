  • Answer #2:
Hello Jerome,

Thanks for reaching out to the Segment support team.

We understand that you wish to make use of Intercom data as a cloud source and add destinations such as Mixpanel and Google analytics.

Unfortunately Intercom Source is an Object source. This means that it sends information (traits) about a thing that exists and persists over time, such as a person or company, and which can be updated over time. Data from this source can only be exported directly to a warehouse, but it can then be used for further analysis.

This meaning adding destinations such as Mixpanel and Google analytics is currently not supported, Warehouses are the only supported destination for object-cloud sources.

  • Question #1:

How many animals live in a Savanna Biome?


  • Answer #1:
Two animals live in the Savanna biome.

Viewing the biomes table, the ‘ID’ column next to Savanna shows, ‘bio52U81’.

Entering the following command:

SELECT animals.biome_id, animals.id

FROM animals

WHERE animals.biome_id = ‘bio52U81’;


The output displays two animals. ‘animDA2GA’ also ‘anim3DK88’.

Pulling both ‘animDA2GA’ and ‘anim3DK88’ from the ‘species’ table via this query:

SELECT species.id, species.name

FROM species

WHERE species.id = ‘animDA2GA’ OR species.id = ‘anim3DK88’;


As seen in the screen shot above, the two animals are Elephants and Cheetahs that live in the Savanna biome.


  • Question #2:

How many biomes are forested?

  • Answer #2:

Three biomes are forested...


By using the… SELECT * FROM biomes; …

We can see that biomes marked ‘true’ or ‘false’ in the fourth column under ‘forested’ is where we need to pull our data.

The following query gives us the answer:

SELECT biomes.name, biomes.forested

FROM biomes

WHERE biomes.forested = ‘t’

    3. Coniferous Forest

  • Question #1:
Describe what the following code block does when executed in a browser
function formatQs() { var output = {}; var qs = document.location.search.substring(1); qs = qs.split('&'); for (var i = 0; i < qs.length; i++) { var tokens=qs[i].split('='); output[tokens[0].toLowerCase()] = tokens[1]; } return output;
  • Answer #1:

Technical:

After researching on the internet, I found out that the Location Search property in HTML DOM is used to set or return the query part of URL including question mark. The query part is the part of URL after the question mark. It will take everything from the ? and to the right.

It then removes the ? with substring(1) and takes the string value and converts it into an array with qs.split('&');

Then it loops through the array and will also loop through values that look like this 'page=1' it uses split again to convert page=1 to this ['page','1'] and then take the second part of this array eg 1 and add it to the variable called output.


Non=technical:

It looks like this is all related to tracking user interactions and search queries on a website.
